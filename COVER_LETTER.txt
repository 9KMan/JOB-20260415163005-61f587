Subject: Amanda AI — SMS Group Trip Planning MVP Proposal

Dear Amanda AI Team,

Group trip planning is broken because it relies on chaotic group chats where preferences get lost, opinions get buried, and nobody wants to be the one who "ruined" the trip by suggesting somewhere unpopular.

Amanda AI solves this by replacing the group chat with a structured SMS survey. The admin creates a trip, shares a link, and each participant privately answers 12 questions via SMS. Nobody sees anyone else's answers. The admin gets an AI-powered summary of the top 1-3 trip options based on actual group preferences.

This is a product with real viral mechanics — every trip has 5-15 participants, each of whom has to engage with SMS, creating organic word-of-mouth. The business model (one-time fee per trip) is simple and defensible.

I've built the MVP that demonstrates this working end-to-end. The code is live at the GitHub link above. Here's what it does:

- Admin creates a trip and pays $49 via Stripe
- System generates a shareable link
- Participants opt in with phone number and consent
- Twilio sends 12 sequential SMS questions
- Each reply is stored privately — nobody sees anyone else's answers
- Admin clicks "Generate Results" and OpenAI produces 1-3 trip recommendations

The MVP is production-ready with one exception: you'll need to add your own Twilio account, Stripe keys, OpenAI key, and Supabase project. I can help with the configuration during the handoff call.
